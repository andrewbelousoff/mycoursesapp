# 📱 MyCoursesApp — Многомодульное Android-приложение курсов

Коммерческое многомодульное Android-приложение для просмотра каталога обучающих курсов и управления избранным. Спроектировано по канонам **Clean Architecture** с использованием реактивного подхода **Single Source of Truth (SSOT)**.

---

## 🛠️ Стек технологий и архитектура

Приложение построено строго по рекомендациям технического задания с использованием современного стека разработки под Android:

*   **Language:** Kotlin + Coroutines (Фоновые потоки через `Dispatchers.IO` для ввода-вывода).
*   **Architecture:** Clean Architecture + MVVM (Model-View-ViewModel).
*   **Dependency Injection:** Koin (Управление графом зависимостей на уровне `Application`).
*   **Local DB:** Room Database (Хранение пользовательских лайков и offline-first кэширование).
*   **Data Parsing:** Gson (Асинхронный парсинг массива данных из локальных ресурсов ассетов).
*   **UI-слой:** Нативный `RecyclerView` + `Fragment` на классической XML-верстке в темных тонах.

---

## 📦 Модуляризация проекта

Кодовая база разделена на независимые функциональные модули для изоляции бизнес-логики и ускорения сборки Gradle:

1.  **`:app`** — Центральный модуль приложения. Отвечает за конфигурацию манифеста, инициализацию DI-графа Koin при старте и верхнеуровневую навигацию (`BottomNavigationView`).
2.  **`:core`** — Изолированный модуль ядра (Domain & Data слои базы данных). Содержит общие сущности моделей (`Course`), интерфейсы репозиториев и компилируемую базу данных Room. Ничего не знает о UI-слоях и других фичах.
3.  **`:feature_courses`** — Изолированный функциональный модуль. Реализует логику отображения списков, поисковую строку, экран детальной информации о курсе и управление состояниями экрана через `StateFlow` во `CoursesViewModel`.

---

## 🧠 Реализация паттерна Single Source of Truth (SSOT)

В проекте отсутствует рассинхронизация данных между окнами благодаря реактивному подходу:
*   При старте приложения данные из `courses.json` выгружаются в фоновом пуле потоков и почленно синхронизируются с базой данных **Room** (`LikedCourseDao`).
*   Экраны «Каталог» и «Избранное» не хранят локальные стейты, а напрямую подписаны на вещание трубы данных `Flow` из репозитория.
*   При клике на кнопку «В избранное» на экране деталей коммит атомарно записывается в базу **Room**, после чего база автоматически обновляет `StateFlow`, и изменения мгновенно отображаются на всех экранах приложения без ручного обновления страниц.

## 🚀 Как запустить проект

1. Клонировать репозиторий:
   ```bash
   git clone https://github.com
   ```
2. Открыть проект в **Android Studio**.
3. Дождаться окончания синхронизации Gradle.
4. Нажать кнопку **Run app** для запуска на эмуляторе или физическом устройстве (Min SDK: 26, Compile SDK: 34).
