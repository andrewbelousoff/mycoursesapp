# MyCoursesApp 📚 — Android Разработка курса каталога

Профессиональное многомодульное Android-приложение для просмотра, фильтрации и управления курсами. Проект разработан строго по техническому заданию (ТЗ) и макетам Figma с использованием современных архитектурных подходов.

## 🛠 Технический Стек (Tech Stack)

- **Language:** Kotlin (JVM 17)
- **Architecture:** Clean Architecture + MVVM + Многомодульность (Multi-module)
- **DI Container:** Koin (с поддержкой общей `activityViewModel()`)
- **Local Database:** Room Database (для хранения локального статуса Избранного)
- **Networking & Parsing:** API Client Architecture / Gson (безопасный парсинг строковых и числовых данных типов)
- **UI & Layout:** ConstraintLayout, CardView, RecyclerView (нативный эффективный адаптер), BottomNavigationView с кастомными селекторами состояний.
- **Asynchrony:** Kotlin Coroutines (Dispatchers.IO / lifecycleScope / repeatOnLifecycle)

---

## 📐 Архитектура проекта (Module Structure)

Проект разделен на независимые слои (модули) для максимальной изоляции бизнес-логики и UI:

1. **`:app`** — Главный модуль. Отвечает за инициализацию DI-контейнера Koin в `MyApplication`, навигацию `MainActivity` (на базе транзакций `add/show/hide` для сохранения состояния экранов) и общие ресурсы темы.
2. **`:core`** — Модуль ядра. Содержит общие доменные модели (`Course`), мапперы слоев данных и локальный источник правды (Room Database / DAO).
3. **`:feature_courses`** — Функциональный модуль. Содержит всю бизнес-логику каталога, `CoursesViewModel`, нативные адаптеры списков, логику двухэтапной стабильной сортировки по датам публикации (`publishDate`) и UI-фрагменты.

## 💡 Ключевые архитектурные решения (Архитектурные Best Practices)

В ходе разработки были успешно решены сложные рантайм-задачи и применены коммерческие паттерны:

- **Single Source of Truth (Единый источник правды):** Сетевые данные из API (имитация через JSON) и локальные лайки из Room Database атомарно синхронизируются на уровне репозитория внутри фонового пула потоков `Dispatchers.IO`. Это исключает рассинхронизацию данных между экранами.
- **Отказоустойчивый парсинг типов данных:** Парсер репозитория защищен от падений типов: поле рейтинга `rate` безопасно переваривает как строковый формат в кавычках (`"4.9"`), так и чистые числа (`4.9`), защищая приложение от скрытых синтаксических исключений.
- **Бессмертная навигация (State Retention):** Переключение вкладок нижнего меню в `MainActivity` реализовано через переключение видимости фрагментов (`show/hide`) вместо их полного уничтожения (`replace`). Это позволило намертво сохранить состояние адаптеров, позицию скролла и текущую сортировку при ходьбе по вкладкам.
- **Двухэтапная стабильная сортировка:** Логика сортировки по дате добавления защищена от хаотичного тасования плиток компаратором `compareByDescending { ... }.thenBy { it.id }`. Курсы без дат (пустые строки в JSON) всегда остаются внизу списка и не прыгают при кликах на кнопки лайков.
- **Ограничение контента по ТЗ:** В разметке карточки `item_course.xml` для описания курса жестко зашито ограничение `maxLines="2"` и `ellipsize="end"`, что гарантирует адаптивность верстки списка на любых экранах.
- **Динамический UI:** Кнопка сортировки динамически меняет свой текст и стрелку (`↑` / `↓`) на лету, а нижнее меню получило сочную цветовую подсветку активной вкладки по Figma через XML Color State селекторы.

## 🚀 Инструкция по запуску (How to Run)

1. Клонируйте репозиторий на локальную машину.
2. Откройте проект в **Android Studio (Ladybug или новее)**.
3. Дождитесь успешного завершения Gradle Sync.
4. Выберите тип сборки **`debug`** (сборка Release требует настройки подписей архивов).
5. Запустите приложение на эмуляторе или реальном устройстве.
6. Для тестирования экрана Входа используются предзаполненные валидные данные: `test@gmail.com` / `12345`.
